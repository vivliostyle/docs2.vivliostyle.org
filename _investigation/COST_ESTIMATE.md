## 2. 開発における追加コスト予測

### 2.1 前提条件

- **使用モデル**: Claude Opus 4.5（プレミアムモデル）
- **開発期間**: 2-3ヶ月
- **推定総トークン**: 80万〜160万トークン
- **人間換算工数**: 80-118時間相当（SSMO対応: Web/PDF/EPUB同時出力）

**注**: 工数は「SSMO（Single Source Multi Output）」アプローチを採用し、1つのソースからWeb/PDF/EPUBを3形式出力する。これにより、3形式別々に開発するより効率的（130-198時間 → 80-118時間）。

### 2.2 GitHub Copilotの課金体系

**Copilot Business（$19/月/シート）に含まれるもの:**
- 通常のコード補完
- GPT-4oベースのチャット
- 月間のプレミアムリクエスト枠

**プレミアムモデル（Claude Opus 4.5等）:**
- 月間枠を超えた場合、従量課金が発生
- レートはGitHubの料金体系に依存

## 6. コスト最適化のヒント

### 6.1 推奨プラクティス

1. **作業の分散**: 月末に集中せず、月初から均等に作業
2. **効率的な指示**: 明確な指示で往復を減らす
3. **モデルの使い分け**: 単純なタスクはGPT-4oで対応
4. **バッチ処理**: 複数ファイルの編集を一度にまとめる
5. **コンテキスト管理**: 必要な情報を事前に整理して提供

### 6.2 監視ポイント

- GitHub Settings → Billing → Usage で月次確認
- Copilot タブでプレミアムリクエストの消費状況を確認
- 月中で予算超過の兆候があれば作業ペースを調整

---

## 7. 注意事項

1. **料金変動**: GitHub Copilotの課金体系は変動する可能性あり
2. **推定の精度**: 上記は推定であり、保証するものではない
3. **定期確認**: 実際の請求額はUsageページで月次確認を推奨
4. **為替影響**: USD建てのため、円換算は為替レートに依存
